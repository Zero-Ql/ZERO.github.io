    ¶Shell命令解释器

    cat /etc/shells

    shell 的命令解释器用于执行 shell 命令或 shell 脚本文件，Centos 下有四种解释器，一般用默认的 /bin/bash 即可

    ¶执行Shell脚本

    要执行 shell 脚本有两种方法：

    1. 将要执行的 .sh 脚本通过 chmod 给予可执行权限，然后执行
    2. 不赋予可执行权限，通过 bash xxx.sh 或 sh xxx.sh 来执行
    chmod +x ./hello.sh    # 方法一

./hello.sh    # 方法二

    使用 .sh 为后缀只是为了方便区分，对实际运行脚本没有任何影响

    ¶Shell变量

    shell 脚本定义变量除了等号两边不能加空格，其他和别的编程语言大差不差（值得一提的是，变量名不支持中文命名，其他大部分编程语言是支持的）

    name="hello world"

    ¶变量类型

    在运行 shell 时，会同时存在三种变量类型：

    1. 局部变量：局部变量在脚本或命令中被定义，只能由当前对象使用，无法被其他 shell 启动的对象访问
    2. 环境变量：环境变量是所有的脚本或程序都能访问的，某些程序或脚本需要环境变量来维持运行，必要时也可以由 shell 脚本自己定义环境变量
    3. shell 变量：shell 变量是由 shell 程序设置的特殊变量，shell 变量中有一部分是环境变量，有一部分是局部变量，这些变量保证了 shell 的正常运行

    ¶使用变量

    要使用定义过的变量，只需在变量名前加美元符号，最好在变量名外加花括号以区分边界

    ¶只读变量

    使用 readonly 可以将变量定义为只读变量，只读变量不能被改变

    readonly name

    ¶删除变量

    使用 unset 删除变量

    unset name

    ¶Shell 传递参数

    有一些特殊变量在 shell 脚本中有特殊含义，如：

    参数处理说明
    $#传递到脚本的参数个数
    $*以一个单字符串的形式输出所有向脚本输入的参数
    $$脚本运行的当前进程 ID 号
    $!后台运行的最后一个进程的ID号
    $@与${*}效果相同，但是加引号时会返回以引号括起来的每个参数，与${*}的区别是${*}将所有参数当一个字符串处理，而${@}则是分别处理每个参数
    $-显示 Shell 使用的当前选项，与 set 命令功能相同
    $?显示最后命令的退出状态。0 表示没有错误，其他任何值都表明有错误
    $0表示脚本的名称
    $1表示脚本的参数，可以有多个，用${1} 、${2} …表示
    #!/bin/bash

echo "有${#}个参数"

echo "参数:${*}"

echo "进程ID:${$}"

echo "最后一个进程ID:${!}"

echo "Shell选项:${-}"

echo "脚本name:${0}"

echo "${1} + ${2} = "`expr ${1} + ${2}`

echo ${?}

    输入：

    bash ./test.sh 10 20

    输出结果：

    有2个参数
    参数:10 20
    进程ID:71819
    最后一个进程ID:
    Shell选项:hB
    脚本name:./test.sh
    10 + 20 = 30
    0

    ¶Shell字符串

    shell 的字符串可以加双引号、单引号，也可以不加引号

    ¶单引号

    • 加单引号的字符串中的变量是无效的，可以使用单引号将变量引用起来达到拼接字符串的目的

    • 单引号字符串中不能出现单独的单引号，转义后也不行，只能成对出现

    ¶双引号

    • 双引号中可以有变量
    • 双引号中可以出现转义字符

    ¶拼接字符串

    使用双引号拼接字符串：

    #!/bin/bash
name="QSky"
echo "Hello! ${name}"

    输出结果：

    Hello! QSky

    输出结果：

    string hello world

    string ${name}

    string hello world

    “string” hello world

    ¶获取字符串的长度

    str="hello world"

echo ${#str}    # ${#str} 等价于 ${#str[0]}

    输出结果：

    11

    ¶提取子字符串

    str="hello world"

# 语法：字符串变量名:要提取的子串的开始位置:要提取的子串的结束位置
echo ${str:6:8}    # 第一个字符 'h' 索引为0

    输出结果：

    world

    ¶查找字符串

    查找字符 i 或 o 的位置，哪个先出现就计算哪个

    str="hello world"
echo `expr index "$str" or`

    输出结果：

    5

    ¶Shell数组

    和大多数编程语言一样，shell 脚本也有数组，但与其他编程语言不同的是，shell 不支持多维数组，不过 shell 的数组并没有长度限制，并且可以使用不连续的下标，下标范围也没有限制

    ¶定义 shell 数组

    数组用小括号来表示，用空格区分值

    格式：

    array_name=(value0 value1 value2 value3)

# 或

array_name=(
value0
value1
value2
value3
)

    示例：

    array_name=(

)

    也能单独定义数组的单个元素

    array_name[0]=value0

array_name[1]=value1

array_name[5]=value5

    ¶读取数组

    和其他编程语言一样，shell 同样是通过数组名加下标来读取元素

    ${数组名[下标]}

    可以使用 @ 或 * 读取数组的全部元素：

    #!/bin/bash
  
str=(
'string '${name}''
'string ${name}'
"string ${name}"
"\"string\" ${name}"
)

echo ${str[@]} # 和 echo ${str[*]} 效果相同

    输出结果：

    ¶为了便于阅读，我给输出结果分段处理了

    string hello world

    string ${name}

    string hello world

    “string” hello world

    ¶获取数组的长度

    获取数组长度的方法与获取字符串长度的方法相似

    #!/bin/bash

array_name=(
value0
value1
value2
value3
)

length_1=${#array_name[@]}

# 或

length_2=${#array_name[*]}

# 获取单个数组元素的长度

length_3=${#array_name[2]}

echo ${length_1} ${length_2} ${length_3}

    输出结果：

    4 4 6

    注：6 为第二个下标所表示的元素的长度

    ¶关联数组

    bash 脚本支持关联数组，可以使用任意的字符串或者整数来作为下标来访问数组。

    关联数组的键是唯一的，有点类似于其他编程语言的键值对。

    语法：

    declare -A array-name=(["key_1"]="value_1"
["key_2"]="value_2"
["key_3"]="value_3"
["key_4"]="value_4"
...
["key_n"]="value_n"
)

    使用 declare 命令来声明，-A 选项就是用于声明一个关联数组。

    #!/bin/bash

declare -A site=(["name"]="QSky"
["url"]="zero-li.fun"
)
echo ${site[@]}

    输出结果：

    zero-li.fun QSky

    同样可以先声明再赋值：

    #!/bin/bash
declare -A site
site["zero"]="zero-li.fun"
echo ${site["zero"]}

    输出结果：

    zero-li.fun

    ¶Shell 注释

    在 shell 脚本中，以 # 开头的就是注释。

    # 这是一句注释
echo 注释的内容并不会被执行

    ¶多行注释

    可以使用 : 命令，并用单引号 ' 将多行内容括起来。

    语法：: + (空格) + ' (单引号)

    : '
这是多段注释
可以有多行内容
'

    shell常用命令及关键字：

    readonly：将变量定义为只读，无法更改其值

    unset：删除变量，无法删除只读变量

    END
